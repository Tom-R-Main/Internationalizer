# Cantonese (yue) — Translation Style Guide

## Language Profile
- Locale: `yue`
- Script: Traditional Chinese (繁體中文), LTR
- CLDR plural forms: other
- Text expansion vs English: -20% to -30%

## Tone & Formality
軟件介面嘅語氣應該保持自然、友善而且專業。翻譯時要用道地嘅廣東話（粵語）口語習慣，避免過度生硬嘅書面語（例如用「嘅」代替「的」，用「喺」代替「在」，用「唔」代替「不」）。對於用戶提示同錯誤訊息，語氣要直接清晰，唔好太過囉嗦。雖然係口語化，但要保持禮貌，唔好用太過粗俗或者太過網絡化嘅潮語。整體感覺要似係一個有禮貌嘅真人同用戶傾偈。

## Grammar
1. **統一粵語用詞**：AI 經常會將書面語（普通話）同廣東話混雜。請確保全句都係廣東話，例如用「唔好」代替「不要」，「撳」代替「點擊」或者「按」，「睇」代替「看」。
2. **按鈕及指令（Imperative forms）**：UI 按鈕應該用簡潔嘅動詞開頭，例如「儲存」、「刪除」。避免加上多餘嘅主語或者助詞。
3. **避免過度使用被動態**：廣東話好少用「被」字。例如 "The file was deleted" 應該譯做「檔案已經刪除」，而唔係「檔案被刪除了」。如果一定要表達被動，可以用「畀」（例如：個檔案畀人刪除咗）。
4. **名詞短語（Labels）**：標籤應該簡短明確。例如 "User Settings" 譯做「用戶設定」，唔需要加「嘅」字（唔好譯做「用戶嘅設定」）。
5. **量詞嘅使用**：廣東話量詞非常豐富而且必須準確。例如 "One error" 係「一個錯誤」，"This photo" 係「呢張相」，"That message" 係「嗰個訊息」。

## Pluralization
廣東話（同其他中文一樣）並無嚴格嘅眾數（Plural）變化，所有數字（包括 0、1、2、100）都對應 CLDR 嘅 `other` 類別。
表達眾數時，通常會喺名詞前面加上數字同量詞，或者用「啲」（例如：呢啲檔案）。
例子：
- 1 file = 1 個檔案
- 3 files = 3 個檔案
- 0 files = 0 個檔案

## Punctuation & Typography
- **標點符號**：必須使用全形標點符號（如：，、。！？）。
- **引號**：主要使用「單引號」，如果引號入面仲有引號，就用『雙引號』。
- **中英夾雜**：英文字母/數字同中文字之間必須加一個半形空格（Space），例如「下載 App 嚟睇下」或者「有 5 個錯誤」。
- **日期格式**：通常用 YYYY-MM-DD（例如：2023-10-25）或者「YYYY年MM月DD日」。
- **時間格式**：多數採用 24 小時制（例如：14:30）或者 12 小時制加朝早/下晝（例如：下晝 2:30）。

## Terminology
| English | Cantonese | Notes |
|---------|-----------|-------|
| Save | 儲存 | 唔好譯做「保存」。 |
| Cancel | 取消 | 標準 UI 用詞。 |
| Delete | 刪除 | 唔好譯做「刪掉」。 |
| Settings | 設定 | 唔好譯做「設置」。 |
| Search | 搜尋 | 亦可以用「搵嘢」，但「搜尋」喺 UI 比較通用。 |
| Error | 錯誤 | 系統出錯時用。 |
| Loading | 載入中 | 唔好譯做「加載中」。 |
| Dashboard | 控制台 | 或者譯做「主頁」，視乎上下文。 |
| Notifications | 通知 | 標準 UI 用詞。 |
| Sign in | 登入 | 唔好譯做「登錄」。 |
| Sign out | 登出 | 唔好譯做「退出」。 |
| Submit | 提交 | 標準 UI 用詞。 |
| Profile | 個人檔案 | 唔好譯做「個人資料」（通常指 Personal Info）。 |
| Help | 說明 | 喺選單通常用「說明」或者「求助」。 |
| Close | 關閉 | 亦可以用「閂」，但按鈕多數用「關閉」。 |
