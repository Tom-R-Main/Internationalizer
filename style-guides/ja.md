# Japanese (ja) — Translation Style Guide

## Language Profile
- Locale: `ja`
- Script: CJK, LTR
- CLDR plural forms: other
- Text expansion vs English: 英語と比較して文字数は約20〜30%減少しますが、全角文字を使用するため実際の表示幅は同等かやや広がる場合があります。

## Tone & Formality
ソフトウェアUIおよび公式ドキュメントでは、標準的な「敬体（です・ます調）」を使用します。プロフェッショナルで礼儀正しく、かつユーザーにとって親しみやすいトーンを保ちます。
- ユーザーにアクションを促す際は、過度に丁寧な表現（例：「〜していただきますようお願い申し上げます」）は避け、簡潔で明確な表現（例：「〜してください」）を使用します。
- エラーメッセージはユーザーを責めるような表現（例：「あなたが間違った入力をしました」）を避け、客観的かつ具体的な解決策を提示するトーンにします。

## Grammar
ソフトウェア翻訳においてLLMが誤りやすい文法規則は以下の通りです。
- **ボタンやメニューのラベル（体言止め）:** 英語の動詞（例: "Save", "Edit"）は、日本語では名詞の体言止め（例: 「保存」「編集」）にします。「保存する」などの動詞形はUIラベルでは避けてください。
- **代名詞の省略:** 「あなた（You）」や「あなたの（Your）」は、文脈上明らかな場合は省略します。直訳すると不自然で冗長な日本語になります。
- **受動態の回避:** 英語の受動態は、可能な限り能動態に変換するか、自然な日本語の表現に意訳してください。（例: "The file was deleted" → 「ファイルが削除されました」または「ファイルを削除しました」）
- **カタカナ語の長音符:** JIS規格の現代的な基準に基づき、末尾の長音符（ー）は原則として省略せず付与します。（例: "User" → 「ユーザ」ではなく「ユーザー」、「Server」 → 「サーバー」）

## Pluralization
日本語には英語のような文法的な単数・複数の区別がありません。CLDRの複数形ルールでは `other` のみが適用されます。
- 数字を含む変数が単数（1）でも複数（2以上）でも、全く同じ翻訳構造を使用します。
- 例: "1 file" / "3 files" → 「1 個のファイル」 / 「3 個のファイル」
- 英語の複数形を示す "s" や "(s)" を「〜ら」や「〜たち」と直訳しないでください。文脈に応じて「複数」と補うか、単に名詞のみを記述します。

## Punctuation & Typography
- **句読点:** 句点には「。」（U+3002）、読点には「、」（U+3001）を使用します。カンマ「,」やピリオド「.」は日本語文脈では原則として使用しません。
- **括弧・引用符:** 強調やUI要素の参照には「」（鉤括弧）を使用します。英語の引用符（"" や ''）は使用しないでください。
- **文字間のスペース:** 半角英数字と全角日本語の間には、原則としてスペースを挿入しません（ベタ組み）。
- **日付と時刻:** 日付は「YYYY年MM月DD日」または「YYYY/MM/DD」の形式を使用します。時刻は24時間表記（例: 14:30）が一般的です。
- **数値の区切り:** 小数点にはピリオド「.」、千の位の区切りにはカンマ「,」を半角で使用します（例: 1,234.56）。

## Terminology
| English | Japanese | Notes |
|---------|----------|-------|
| Save | 保存 | ボタンやメニューのラベルとして使用（体言止め）。 |
| Cancel | キャンセル | 「取消」よりも一般的で自然な表現。 |
| Delete | 削除 | データやファイルを消去するアクション。 |
| Settings | 設定 | アプリケーションやアカウントの設定画面・メニュー。 |
| Search | 検索 | 検索ボタンや入力フィールドのプレースホルダーに使用。 |
| Error | エラー | 障害や問題の発生を示す。 |
| Loading | 読み込み中 | 進行中の状態を示す。「ロード中」よりも標準的。 |
| Dashboard | ダッシュボード | カタカナ表記が標準的。 |
| Notifications | 通知 | ユーザーへのシステム通知やアラート。 |
| Sign in | サインイン | ログイン（Login）と混同しないよう、原文の語彙に従う。 |
| Sign out | サインアウト | ログアウト（Logout）と混同しないよう、原文の語彙に従う。 |
| Submit | 送信 | フォームの送信ボタンなどに使用。「提出」は文脈を選ぶため注意。 |
| Profile | プロフィール | 「プロファイル」ではなく「プロフィール」がユーザー向けUIでは一般的。 |
| Help | ヘルプ | サポートやドキュメントへのリンク。 |
| Close | 閉じる | モーダル、ダイアログ、ウィンドウを閉じるアクション。 |
